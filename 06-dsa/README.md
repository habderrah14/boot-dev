# Module 06 — Learn Data Structures and Algorithms in Python

> Data structures are the vocabulary. Algorithms are the grammar. Without them, every backend problem looks the same — and your solutions will all perform the same, which is to say: badly.

## Map to Boot.dev

Parallels Boot.dev's **"Learn Data Structures and Algorithms"** course, chapter-for-chapter.

## Learning outcomes

By the end of this module you will be able to:

- Analyze any algorithm you write or read using Big-O notation.
- Implement stacks, queues, linked lists, hashmaps, binary trees, red-black trees, tries, and graphs from scratch.
- Choose the right structure for a problem based on access patterns, not familiarity.
- Implement and compare sorting algorithms (bubble, insertion, merge, quick) and know why `sort()` in the standard library beats them.
- Perform BFS and DFS on a graph and explain when each is appropriate.
- Articulate what **P vs NP** actually says — and why "NP-hard" does not mean "impossible".

## Quiz answer key

- **Ch. 01 — Algorithms Intro.** 1) b, 2) a, 3) b, 4) a, 5) b.
  - 6.  Linear search can beat binary search when the data is tiny, unsorted, or searched only once, because sorting would cost more than scanning.
  - 7.  Correctness, performance, simplicity, generality.
- **Ch. 02 — Math.** 1) b, 2) b, 3) c, 4) a, 5) b.
  - 6.  Modulo maps large hash values into valid index ranges for fixed-size tables.
  - 7.  Common uses include rate limiting windows, token/session generation, round-robin balancing, and circular buffers.
- **Ch. 03 — Big-O Analysis.** 1) b, 2) b, 3) b, 4) b, 5) b.
  - 6.  Dominant terms matter because lower-order terms become negligible as input size grows.
  - 7.  Constants matter for small n, same-class comparisons, and expensive operations like network or disk I/O.
- **Ch. 04 — Sorting Algorithms.** 1) c, 2) a, 3) a, 4) b, 5) b.
  - 6.  Stability is critical for multi-key sorting because it preserves earlier key order.
  - 7.  Insertion sort often wins on tiny arrays due to lower overhead and better locality.
- **Ch. 05 — Exponential Time.** 1) b, 2) b, 3) c, 4) b, 5) b.
  - 6.  Memoization removes redundant recomputation by caching each subproblem once.
  - 7.  Practical NP-hard tactics include heuristics, approximation algorithms, branch-and-bound, and solver-based methods.
- **Ch. 06 — Data Structures Intro.** 1) c, 2) b, 3) b, 4) b, 5) d.
  - 6.  Every structure trades strengths and weaknesses across operations; no universal O(1) structure exists for all operations.
  - 7.  Linked lists fit scenarios with O(1) insertion/deletion at known node positions (often paired with a hashmap).
- **Ch. 07 — Stacks.** 1) b, 2) c, 3) a, 4) b, 5) b.
  - 6.  Balanced-parentheses validation pushes opens and pops on matching closes until stack empties cleanly.
  - 7.  Common uses: undo/redo, DFS, postfix evaluation, browser history, and call-stack modeling.
- **Ch. 08 — Queues.** 1) b, 2) b, 3) b, 4) c, 5) a.
  - 6.  BFS uses queue FIFO order to process nodes by increasing distance/level.
  - 7.  Heaps beat sorted lists for streaming minimum/maximum extraction with dynamic insertions.
- **Ch. 09 — Linked Lists.** 1) c, 2) a, 3) b, 4) a, 5) b.
  - 6.  LRU caches use hashmap + doubly linked list for O(1) lookup and move-to-front updates.
  - 7.  In-place reversal uses constant auxiliary space because it only tracks a few pointers.
- **Ch. 10 — Binary Trees.** 1) b, 2) b, 3) b, 4) b, 5) b.
  - 6.  In-order traversal of a BST is sorted because it visits left subtree, node, then right subtree.
  - 7.  Degenerate BSTs collapse to O(n) operations; self-balancing trees prevent that.
- **Ch. 11 — Red-Black Trees.** 1) b, 2) b, 3) c, 4) b, 5) b.
  - 6.  Rotations are local structural changes that restore balance while preserving BST ordering.
  - 7.  Prefer hashmaps when sorted order/range queries are unnecessary and average O(1) lookup is desired.
- **Ch. 12 — Hashmaps.** 1) a, 2) b, 3) b, 4) b, 5) a.
  - 6.  Mutable keys break lookup consistency because their hash/bucket mapping changes after insertion.
  - 7.  Collision attacks can degrade expected O(1) behavior toward O(n) (hash-flooding DoS).
- **Ch. 13 — Tries.** 1) c, 2) b, 3) d, 4) b, 5) a.
  - 6.  Tries support efficient prefix/autocomplete queries without scanning all keys.
  - 7.  Radix compression merges single-child chains, reducing nodes and memory usage.
- **Ch. 14 — Graphs.** 1) b, 2) b, 3) b, 4) c, 5) b.
  - 6.  Adjacency matrices are strong for dense graphs and constant-time edge-existence queries.
  - 7.  Dependency graphs model services/components for cycle detection and deployment ordering.
- **Ch. 15 — BFS and DFS.** 1) b, 2) b, 3) b, 4) b, 5) c.
  - 6.  DFS naturally supports topological sort and directed cycle detection via recursion-state tracking.
  - 7.  Queue vs stack determines exploration order: breadth-first levels vs depth-first branch exploration.
- **Ch. 16 — P vs NP.** 1) a, 2) b, 3) b, 4) b, 5) c.
  - 6.  NP-complete problems are in NP and NP-hard; NP-hard problems may be outside NP entirely.
  - 7.  NP-hard still allows practical progress via structure-aware heuristics and approximation methods.
