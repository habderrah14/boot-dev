# Module 11 · Extended Assessment — System Design Prompt

## 1) Warm-up — design vocabulary

Define consistency, isolation, replication lag, hot partition, and index selectivity with one backend example each.

## 2) Standard — design persistent model

Design storage for one service (notes, inventory, billing): entities, indexes, constraints, and migration path.

## 3) Bug hunt — scale breakpoints

Identify three likely breakpoints at 10x scale and propose mitigations.

## 4) Stretch — capacity memo (≤300 words)

Estimate p95 query latency budget and dominant bottleneck using explicit assumptions.

## 5) Stretch++ — peer critique

Critique a peer schema and require one pre-production change.

**Takeaway:** Database design is architecture, not just DDL.
