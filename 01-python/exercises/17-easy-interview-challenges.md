# Module 01 · Extended Assessment — Easy Interview Challenges

Practice concise problem-solving with fundamentals only (Modules 01–05 scope).

## 1) Warm-up — five-minute circuit

Solve 5 micro-prompts in 5 minutes each:

- count char frequency,
- second-largest value,
- palindrome list check,
- first duplicate,
- safe numeric parse.

**Success check:** All solutions run and at least 4/5 are correct on first test.

**What you should have learned:** Speed improves when patterns are recognized early.

## 2) Standard — two easy interview problems

Solve two easy problems from a public platform or provided prompt list.
For each include:

- final code,
- complexity,
- 4 edge cases.

Save to `practice/17-easy-interview-challenges/solutions.md`.

**Success check:** Both solutions pass your edge cases.

**What you should have learned:** Interview code is judged on thinking and correctness under constraints.

## 3) Bug hunt — break your own solution

Take one solution from step 2 and intentionally inject 2 bugs.
Then debug and fix both while documenting the process.

**Success check:** You can explain the failure signal that led to each fix.

**What you should have learned:** Interview resilience means recovering from mistakes quickly.

## 4) Stretch — whiteboard explanation (≤200 words each)

For each solved problem, write a plain-language explanation as if teaching a non-programmer.

**Success check:** Explanation avoids jargon and still preserves algorithm correctness.

**What you should have learned:** Clear communication is part of interview performance.

## 5) Stretch++ — timed mock round

Run a 25-minute mock:

- 3 min clarifying questions,
- 15 min implementation,
- 7 min test + explain tradeoffs.

Record what you would improve in your next round.

**Success check:** You finish within time and call out at least one concrete improvement.

**What you should have learned:** Process discipline is a competitive advantage in interviews.
