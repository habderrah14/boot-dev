# Mini-project — 03-github-profile

_Companion chapter:_ [`03-github-profile.md`](../03-github-profile.md)

**Goal.** Ship a profile that a friend in tech would confidently forward to a hiring manager.

**Acceptance criteria.**

- Real name, professional avatar, one-sentence bio, contact links.
- Profile README exists and lists your headline + depth projects with one-line pitches.
- 3–6 pinned repos, each with a description, topics, homepage link (if applicable), and a complete README.
- Public repo count halved (or more) from the starting count via archiving and privatizing.
- A trusted friend's 30-second review says: "yes, this reads like a junior backend engineer."

**Hints.** Do the audit in one sitting (1 hour). Archive aggressively — you can always un-archive. Ask your friend to review in incognito mode so they see what a stranger sees.

**Stretch.** Run a GitHub secrets scanner (like `trufflehog` or `gitleaks`) on all your public repos before publishing.
