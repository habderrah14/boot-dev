# Module 01 — Learn to Code in Python

> Your first programming language. Python is used everywhere in backend work — from quick scripts to massive production services — and its clean syntax makes it the best starting point for someone who wants to understand programming itself, not just memorize ceremonies.

## Map to Boot.dev

This module parallels the Boot.dev course **"Learn to Code in Python"**. Every chapter here corresponds 1:1 to a Boot.dev lesson chapter.

## Learning outcomes

By the end of this module you will be able to:

- Read and write Python 3.12+ code comfortably using variables, functions, and the basic control-flow constructs.
- Choose the right built-in data structure (list, dict, set) for a given problem and explain the trade-off.
- Debug a misbehaving program using `print`, the interactive REPL, and an exception traceback.
- Write and run your own `unittest` tests, and recognize what "testable" code looks like.
- Translate an English-language problem statement into a correct, readable Python program.

## Prerequisites

- A computer with Python 3.12 or later installed. On Linux/macOS `python3 --version`; on Windows install from python.org or the Microsoft Store.
- A text editor. VS Code is recommended for first-timers because of its built-in terminal.
- No prior programming experience is assumed.

## How this module connects

- _Variables, functions, scope_ reappear in every language: JavaScript, TypeScript, C.
- _Lists, dicts, sets_ are the concrete warm-up for the abstract **data structures** of Module 06.
- _Testing & debugging_ is the foundation for everything that follows; expect to lean on it through the entire path.
- _Errors_ sets up exception handling that you'll see again in HTTP servers (Module 12) and message brokers (Module 15).
