# Module 02 · Extended Assessment — Debugging Incident Lab

## 1) Warm-up — triage shell failures

Identify cause and first command to investigate for three terminal/file-permission failures.

## 2) Standard — incident report

Create `practice/07-debugging-incident-lab/postmortem.md` with timeline, root cause, fix, prevention.

## 3) Bug hunt

Fix three failures: broken path assumption, permission denied, wrong redirection behavior.

## 4) Stretch

Write a 200-word runbook for "script fails in CI but not locally".

## 5) Stretch++

Run a peer simulation and capture one runbook improvement.

**Takeaway:** Linux debugging is reproducible when diagnosis is command-driven.
