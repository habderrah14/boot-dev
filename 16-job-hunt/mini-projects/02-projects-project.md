# Mini-project — 02-projects

_Companion chapter:_ [`02-projects.md`](../02-projects.md)

**Goal.** Publish or republish one headline project with an interview-ready README.

**Acceptance criteria.**

- A live demo URL (deployed) or a recording if deployment isn't feasible.
- README with all 8 sections: one-liner, demo, why, architecture, key decisions, setup, tests, roadmap.
- `git log` shows 20+ commits, including at least one refactor commit.
- A non-technical friend can read the top of the README and state what the project does.
- A friend in the target role can read the "decisions" section and ask at least one follow-up question.

**Hints.** If your project isn't deployed yet, use Fly.io, Railway, or Render. If it's a CLI tool, record a terminal session with `asciinema` as your demo.

**Stretch.** Write a companion blog post explaining one key decision in depth. Link it from the README.
